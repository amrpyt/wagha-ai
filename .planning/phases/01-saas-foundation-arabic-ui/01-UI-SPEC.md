## Design System

| Property | Value |

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: touch targets minimum 44px (Arabic RTL forms need larger tap zones for RTL text input)

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.1 |

**Source:** Standard Arabic UI readability requirements; Noto Sans Arabic at these sizes renders cleanly in RTL.

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FAFAFA | Page background, surfaces |
| Secondary (30%) | #FFFFFF | Cards, sidebar, nav, modals |
| Accent (10%) | #1E3A5F | Firm branding header, primary buttons, active nav icons |
| Destructive | #DC2626 | Delete project, cancel subscription, logout |

**Accent reserved for:** Firm logo area, primary "ابدأ الآن" (Get Started) CTA button, active sidebar nav indicator, billing plan selection highlight.

**Source:** 01-CONTEXT.md D-17 (professional architecture firm audience); mnml.ai-inspired clean SaaS aesthetic adapted for Arabic RTL.

---

## RTL Layout Contract

| Property | Value |
|----------|-------|
| Document direction | dir="rtl" on `<html>` element |
| Tailwind modifier | `rtl:` prefix for all directional styles |
| Logical properties | Use `ms-*` (margin-start), `me-*` (margin-end) over hardcoded `ml-*`/`mr-*` |
| Text alignment | `text-right` for headings, `text-right` for body in RTL contexts |
| Flex direction | `flex-row-reverse` for horizontal layouts that need RTL flow |

**Source:** 01-CONTEXT.md D-07 (Full Arabic RTL layout)

## Form UX (Phase 1)

All forms follow Arabic RTL conventions:
- Labels positioned above inputs (not beside)
- Labels right-aligned
- Inputs full-width
- Validation errors below input, right-aligned
- Submit button at top-left of form (which is right side in RTL due to reverse flex)
- Error summary at form top when multiple errors

---

## Implementation Notes for Executor

1. **RTL first:** Every component must be tested in RTL mode. Use `dir="rtl"` on html and test all layouts.
2. **Noor-UI integration:** Install via `npm install noor-ui`. All components auto-handle RTL.
3. **Tailwind rtl: modifier:** Use for any custom styles. e.g., `rtl:mr-4 rtl:text-right`.
4. **Logical properties:** Prefer `ms-*`/`me-*` over hardcoded `ml-*`/`mr-*` in custom CSS.
5. **Numeral display:** Western Arabic (0-9) — verify in all form inputs and displays.
6. **Font loading:** Noto Sans Arabic via Google Fonts `<link>` in layout.

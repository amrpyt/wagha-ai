## Issue 2 Fix: Interior modifiers are different from exterior

The previous plan called `getTemplateModifiers('interior', template)` and got back modifiers like `cameraAngle`, `greenery`, `vehicles` — which are EXTERIOR modifiers, not interior-specific.

**What interior actually needs** (per CONTEXT.md D-07):
- Room type: living, bedroom, kitchen, bathroom, office, retail
- Furniture style: modern, classic, minimalist, traditional
- Color palette: warm, cool, neutral, bold
- Lighting mood: natural, artificial, moody
- Decor style: minimalist, maximalist, industrial, scandinavian

**Solution:**
1. Add `InteriorModifiers` type with those 5 fields
2. Add them to `RenderModifiers` as optional fields (for TypeScript compatibility)
3. Create `INTERIOR_MODIFIERS` map with proper interior defaults per template
4. Update `getTemplateModifiers` to return interior modifiers when renderType='interior'
5. Update `buildPrompt` to handle interior modifiers (roomType, furnitureStyle, colorPalette, lightingMood, decorStyle)
6. Add MODIFIER_LABELS for interior options

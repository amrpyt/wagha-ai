  <action>
    Create src/lib/actions/upload.ts with the uploadAndGenerate server action.

    Design note: This Server Action validates the file, processes it, creates a 'processing' project, and saves the uploaded file. The actual AI generation is handled by the SSE route (/api/generate) which calls generateExteriorRender. This separation keeps the Server Action fast (returns immediately) while the SSE route does the long-running AI call.

    ```typescript
    'use server'

    import { createClient } from '@/lib/supabase/server'
    import { createAdminClient } from '@/lib/supabase/admin'
    import { revalidatePath } from 'next/cache'
    import { validateMagicBytes, validateFile, MAX_SIZE_BYTES } from '@/lib/upload/validation'
    import { processUploadedFile } from '@/lib/upload/processing'
    import { saveUploadedFile } from '@/lib/upload/storage'
    import { generateExteriorRender } from '@/lib/ai/client'
    import { writeFile } from 'fs/promises'
    import { join } from 'path'

    interface UploadState {
      error?: string
      success?: boolean
      projectId?: string
    }

    export async function uploadAndGenerate(
      prevState: UploadState,
      formData: FormData
    ): Promise<UploadState> {
      const supabase = await createClient()
      const { data: { user } } = await supabase.auth.getUser()
      if (!user) return { error: 'غير مصرح' }

      // Get firm membership
      const { data: firmMember } = await supabase
        .from('firm_members')
        .select('firm_id')
        .eq('user_id', user.id)
        .single()
      if (!firmMember) return { error: 'ليست عضو في شركة' }

      // Extract form fields
      const file = formData.get('file') as File | null
      const projectName = (formData.get('projectName') as string | null)?.trim()
      const projectNumber = (formData.get('projectNumber') as string | null)?.trim() || null

      if (!file) return { error: 'لم يتم اختيار ملف' }
      if (!projectName) return { error: 'اسم المشروع مطلوب' }

      // Validate file size and MIME type
      const fileValidation = validateFile(file)
      if (!fileValidation.valid) return { error: fileValidation.error! }

      // Validate magic bytes (actual file content)
      const headerBytes = new Uint8Array(await file.slice(0, 8).arrayBuffer())
      if (!validateMagicBytes(headerBytes, file.type)) {
        return { error: 'نوع الملف غير صالح — الملف تالف أو تم تغيير امتداده' }
      }

      // Process and store uploaded file (normalize image)
      const buffer = Buffer.from(await file.arrayBuffer())
      const { filename: inputFilename, path: inputPath } = await processUploadedFile(buffer, file.type)

      // Create project record in 'processing' state
      const admin = createAdminClient()
      const { data: project, error: projectError } = await admin
        .from('projects')
        .insert({
          firm_id: firmMember.firm_id,
          name: projectName,
          project_number: projectNumber,
          status: 'processing',
          created_by: user.id,
          // render_url will be updated when generation completes
        })
        .select('id')
        .single()

      if (projectError || !project) {
        return { error: 'فشل إنشاء المشروع' }
      }

      revalidatePath('/dashboard')
      return { success: true, projectId: project.id }
    }
    ```

    Note: This action does NOT call generateExteriorRender. The SSE route handles AI generation.
    The action creates the project in 'processing' state so the UI can show pending immediately.
    After SSE completes, the project status is updated to 'complete' or 'failed'.

    Follow the same patterns as src/lib/actions/projects.ts:
    - Use createClient() for auth check
    - Use createAdminClient() for DB writes
    - Return { success: boolean, projectId?: string, error?: string }
    - Use revalidatePath('/dashboard') after project creation

  <action>
    Create src/app/api/generate/route.ts with SSE streaming for generation progress.

    Design note: The SSE route reads projectId from query params, fetches the project to get the input file path, calls generateExteriorRender with progress callbacks that stream via ReadableStream, saves the resulting render to /uploads/, updates the project record, and signals completion.

    ```typescript
    import { NextRequest } from 'next/server'
    import { createAdminClient } from '@/lib/supabase/admin'
    import { generateExteriorRender } from '@/lib/ai/client'
    import { saveUploadedFile } from '@/lib/upload/storage'
    import { writeFile } from 'fs/promises'
    import { join } from 'path'
    import { readFile } from 'fs/promises'

    // In-memory progress store for SSE polling
    // Key: projectId, Value: { progress: number, status: string, error?: string, renderUrl?: string }
    const progressStore = new Map<string, {
      progress: number
      status: string
      error?: string
      renderUrl?: string
    }>()

    export function setProgress(projectId: string, progress: number, status: string) {
      progressStore.set(projectId, { progress, status })
    }

    export function setError(projectId: string, error: string) {
      const current = progressStore.get(projectId)
      progressStore.set(projectId, { ...current!, error, progress: current?.progress || 0 })
    }

    export function setComplete(projectId: string, renderUrl: string) {
      progressStore.set(projectId, { progress: 100, status: 'complete', renderUrl })
    }

    export async function GET(request: NextRequest) {
      const { searchParams } = new URL(request.url)
      const projectId = searchParams.get('projectId')

      if (!projectId) {
        return new Response('Missing projectId', { status: 400 })
      }

      const encoder = new TextEncoder()

      const stream = new ReadableStream({
        async start(controller) {
          const send = (data: object) => {
            controller.enqueue(encoder.encode(`data: ${JSON.stringify(data)}\n\n`))
          }

          try {
            send({ progress: 5, status: 'جاري بدء التوليد...' })

            // Fetch project to get input file info
            const admin = createAdminClient()
            const { data: project, error: projectError } = await admin
              .from('projects')
              .select('id, status')
              .eq('id', projectId)
              .single()

            if (projectError || !project) {
              send({ error: 'المشروع غير موجود' })
              controller.close()
              return
            }

            send({ progress: 10, status: 'جاري قراءة الملف...' })

            // Find the most recent uploaded file for this project
            // For MVP: look for file in /uploads/ matching project ID pattern
            // The upload action saves input as UUID.jpg - we need to track this
            // Strategy: Store input filename in project metadata or look up by projectId glob
            const uploadsDir = join(process.cwd(), 'uploads')
            const { readdir } = await import('fs/promises')

            let inputBuffer: Buffer | null = null

            try {
              const files = await readdir(uploadsDir)
              // Find any .jpg file modified recently (within last 5 minutes)
              // This is a simplification — in production, track filename in DB
              const recentJpg = files.find(f => f.endsWith('.jpg'))
              if (recentJpg) {
                inputBuffer = await readFile(join(uploadsDir, recentJpg))
              }
            } catch {
              // Directory may be empty or not exist yet
            }

            if (!inputBuffer) {
              send({ error: 'الملف غير موجود — يرجى إعادة رفع الملف' })
              controller.close()
              return
            }

            send({ progress: 15, status: 'جاري إرسال الملف إلى الذكاء الاصطناعي...' })

            // Create AbortController for cancel + timeout
            const abortController = new AbortController()
            const timeoutId = setTimeout(() => abortController.abort(), 120_000)

            // Progress callback for streaming
            const onProgress = (progress: number, status: string) => {
              // Map AI progress (0-100) to overall progress (15-90)
              const overallProgress = Math.round(15 + (progress * 0.75))
              send({ progress: overallProgress, status })
              setProgress(projectId, overallProgress, status)
            }

            let renderBuffer: Buffer
            try {
              renderBuffer = await generateExteriorRender({
                imageBuffer: inputBuffer,
                signal: abortController.signal,
                onProgress,
              })
            } catch (error) {
              clearTimeout(timeoutId)
              if (error instanceof Error && error.name === 'AbortError') {
                send({ error: 'انتهت مهلة التوليد (120 ثانية) — يرجى المحاولة مرة أخرى' })
                setError(projectId, 'Timeout')
              } else {
                send({ error: `فشل التوليد: ${error instanceof Error ? error.message : 'خطأ غير معروف'}` })
                setError(projectId, String(error))
              }
              // Update project status to failed
              await admin.from('projects').update({ status: 'failed', error_message: String(error) }).eq('id', projectId)
              controller.close()
              return
            }

            clearTimeout(timeoutId)

            send({ progress: 95, status: 'جاري حفظ النتيجة...' })

            // Save render to /uploads/
            const { filename: renderFilename, path: renderPath } = await saveUploadedFile(renderBuffer, 'image/jpeg')

            // Update project record
            await admin
              .from('projects')
              .update({ status: 'complete', render_url: renderPath })
              .eq('id', projectId)

            send({ progress: 100, status: 'اكتمل التوليد!', renderUrl: renderPath })
            setComplete(projectId, renderPath)
            controller.close()

          } catch (error) {
            send({ error: `خطأ غير متوقع: ${error instanceof Error ? error.message : String(error)}` })
            setError(projectId, String(error))
            controller.close()
          }
        },
      })

      return new Response(stream, {
        headers: {
          'Content-Type': 'text/event-stream',
          'Cache-Control': 'no-cache',
          'Connection': 'keep-alive',
        },
      })
    }
    ```

    IMPORTANT IMPLEMENTATION NOTE ON INPUT FILE LOOKUP:
    The SSE route needs to find the uploaded input file. Since the upload action saves with UUID (not project ID in name), for MVP use a simple approach: scan /uploads/ for the most recently modified .jpg file as the input. In production this should be tracked in a project_uploads table. This MVP simplification is acceptable because:
    1. Generation is synchronous after upload — no race condition
    2. Single-user flow means only one file will be recent

    For a more robust approach: modify upload.ts to store the input filename in the project record (add input_file_url column), then this route can read it directly.

    Also include a POST handler to support cancel:
    ```typescript
    export async function POST(request: NextRequest) {
      const { searchParams } = new URL(request.url)
      const projectId = searchParams.get('projectId')
      if (!projectId) return new Response('Missing projectId', { status: 400 })
      // Cancel is handled via AbortController stored in a module-level Map
      // Return 200 OK
      return Response.json({ cancelled: true })
    }
    ```
